# Brand Voice — template

> **How to use this file.** Voice is _how_ you say what the [Brand Soul](./brand_soul.md) decided you are. Fill each section with concrete rules and real before/after rewrites — examples teach voice far better than adjectives. The most useful part of this doc is the **do/don't table** and the **rewrites**; spend your time there. Keep it to rules a new writer (or an AI agent) could apply without asking you.

## 3. Tone by context

_Same voice, dial shifts with the moment. Define the dial._

| Context               | Dial toward               | Avoid                         |
| --------------------- | ------------------------- | ----------------------------- |
| Landing / marketing   | confident, benefit-first  | hype, superlatives            |
| Docs / how-to         | plain, imperative, calm   | cleverness, filler            |
| Errors / empty states | honest, helpful, no blame | cutesy, apologetic-to-a-fault |
| Changelog / release   | matter-of-fact, specific  | breathless "🎉 HUGE" energy   |

## 4. Rules of thumb

_Hard rules that make copy sound like us. Keep them enforceable._

- Prefer the **shortest word that's still precise**. Cut "in order to," "utilize," "leverage."
- **Verbs over nouns.** "Ship it" > "facilitate deployment."
- One idea per sentence. No staccato-for-effect ("Three. Short. Sentences.") — that reads as AI. Flow over chop.
- Say the tradeoff out loud. Trust comes from honesty, not polish.
- Second person ("you"), active voice, present tense.
- No manufactured tagline closers. Real people don't end paragraphs with a poster slogan.

## 5. Vocabulary

_Words we use, words we ban, and how we spell our own terms._

- **Use:** ship, build, role, map, service, lib, adapter …
- **Ban:** synergy, revolutionary, seamless, cutting-edge, "simply/just" (as in "just do X") …
- **Our terms (spell consistently):** `apps` / `services` / `libs` (lowercase, in code font), Tilt, `@stack/*` …
